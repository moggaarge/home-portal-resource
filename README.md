# home-portal-resource

## 项目简介

本仓库用于归档和发布多个独立的 HTML 页面。这些页面各自保持完整，不依赖特定域名或网站环境，可直接在浏览器中打开使用。仓库的主要目的是集中存放各类静态页面资源，便于索引、维护和版本管理。

## 目录说明

```
home-portal-resource/
├── pages/          # 存放所有独立 HTML 页面
├── assets/         # 页面引用的公共资源（如样式、脚本、图片等）
└── README.md       # 本文件
```

- `pages/`：每个子目录或文件对应一个独立的页面，页面之间互不依赖。
- `assets/`：存放页面可能共享的静态资源，可按照类型或用途进一步组织。

## 页面归档说明

- 每个页面均为完整的 HTML 文件，包含必要的样式和脚本。
- 页面内容不针对特定网站或域名，保持通用性。
- 归档的页面可能涵盖工具、信息展示、实验性设计等不同类型。
- 页面命名应体现其用途，便于识别。

## 维护说明

- 新增页面：在 `pages/` 下创建新目录或文件，并确保其独立可运行。
- 更新页面：直接修改对应文件，保持向后兼容或注明变更。
- 资源管理：公共资源放入 `assets/`，避免在页面内重复内联。
- 提交信息应简洁描述变更内容，如“新增 xx 页面”或“更新 xx 页面样式”。

## 注意事项

- 本仓库不托管动态内容或后端逻辑。
- 页面不应包含任何敏感信息或外部依赖（除非在 `assets/` 中明确引用）。
- 欢迎提交 Issue 或 Pull Request 以改进页面质量或增加新类型。
